我在deepin下的vscode配置

个人偏爱使用 clang+xmake，以下内容均针对这一配置

首先配置clang环境，deepin 20 clang版本过低的解决方案见这里

接下来安装插件：

  • clangd（提供代码补全，格式化等功能）
  • CodeLLDB（调试）
  • Project Manager（管理项目）
  • XMake (xmake的vscode插件)

https://xmake.io/ 是一个很好用的构建工具，用 lua 描述配置，语法简单，功能强大，帮你远离 CMake 的折磨。

官网提供的安装方式：

最后把vscode的插件配置一下就行了。

Xmake: Debug Config Type 改为 codelldb
Clangd: Path 改为 clangd-15

可使用 xmake create <project-name> 创建项目，vscode打开。效果如下：
